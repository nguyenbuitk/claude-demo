Create `.github/workflows/ci.yml` with the following exact structure (per D-01, D-04):

**Top-level name:** `CI`

**Triggers (per D-02):**
```yaml
on:
  pull_request:
  push:
    branches: [main]
```

**Job 1: `test`** (per D-02, CI-01):
- `runs-on: ubuntu-latest`
- Steps:
  1. `actions/checkout@v4`
  2. `actions/setup-python@v5` with `python-version: '3.12'` and `cache: 'pip'` (discretion: pip caching via setup-python built-in)
  3. `pip install -r requirements.txt` and `pip install pytest` (pytest is a dev dependency, not in requirements.txt)
  4. `pytest` to run all tests

**Job 2: `build-and-push`** (per D-03, D-08, CI-02):
- `needs: [test]` — blocked by test failure (D-08)
- `if: github.ref == 'refs/heads/main' && github.event_name == 'push'` — never runs on PRs (D-03)
- `runs-on: ubuntu-latest`
- **Permissions block** (per D-07, CI-04):
  ```yaml
  permissions:
    contents: read
    packages: write
  ```
- Steps:
  1. `actions/checkout@v4`
  2. `docker/login-action@v3` with:
     - `registry: ghcr.io`
     - `username: ${{ github.actor }}`
     - `password: ${{ secrets.GITHUB_TOKEN }}`
  3. `docker/metadata-action@v5` with:
     - `images: ghcr.io/${{ github.repository_owner }}/claude-demo` (per D-05, CI-03)
     - `tags:` block containing two tag rules:
       ```yaml
       tags: |
         type=raw,value=latest
         type=sha,prefix=sha-,format=short
       ```
       This produces `:latest` and `:sha-a1b2c3d` (7-char short SHA) per D-06.
  4. `docker/build-push-action@v6` with:
     - `context: .`
     - `push: true`
     - `tags: ${{ steps.meta.outputs.tags }}`
     - `labels: ${{ steps.meta.outputs.labels }}`
     The metadata step must have `id: meta` for this reference to work.

**Do NOT modify** `.github/workflows/claude.yml` or `.github/workflows/claude-code-review.yml` (per CONTEXT.md canonical_refs).
