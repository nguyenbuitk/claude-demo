<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: Verify PR triggers test job only, then merge and verify GHCR image</name>

  <read_first>
    - .planning/phases/02-docker-ci-pipeline/02-RESEARCH.md (Validation Architecture > Live Validation section for exact steps)
    - .github/workflows/ci.yml (the workflow being verified)
  </read_first>

  <what-built>
    Complete CI pipeline in `.github/workflows/ci.yml` with:
    - `test` job: runs pytest on every PR and push to main
    - `build-and-push` job: builds Docker image and pushes to GHCR, only on push to main, only after test passes
  </what-built>

  <how-to-verify>
    **Step 1 — PR Check (verifies CI-02, CI-04):**
    1. Create a branch and open a PR containing `.github/workflows/ci.yml`
    2. In the PR checks UI, confirm:
       - "CI / test" appears as a check and runs
       - "CI / build-and-push" does NOT appear (or shows as skipped)
       - "Tests / test" from test.yml also appears (both workflows fire)

    **Step 2 — Merge Check (verifies CI-01, CI-03, CI-05, CI-06, CI-07):**
    1. Merge the PR to main
    2. Go to Actions tab, find the "CI" workflow run triggered by the push
    3. Confirm `test` job ran first and passed
    4. Confirm `build-and-push` job ran after test and passed
    5. Navigate to https://github.com/nguyenbuitk?tab=packages
    6. Find the `claude-demo` package
    7. Confirm two tags exist:
       - `latest`
       - `sha-<7-char-sha>` matching the merge commit (`git rev-parse --short HEAD`)

    **Step 3 — Auth verification (verifies CI-07):**
    1. Confirm no secrets were manually created in repo Settings > Secrets
    2. The workflow uses only `secrets.GITHUB_TOKEN` (automatic)

    **Optional Step 4 — Package visibility:**
    1. If desired, set the claude-demo package visibility to Public at https://github.com/nguyenbuitk?tab=packages > claude-demo > Package settings
  </how-to-verify>

  <resume-signal>
    Type "approved" if all checks pass, or describe which step failed and the error message.
  </resume-signal>

  <acceptance_criteria>
    - PR checks UI shows "CI / test" running
    - PR checks UI does NOT show "CI / build-and-push" running (skipped or absent)
    - After merge: Actions tab shows test passed then build-and-push passed
    - GHCR package page shows `latest` tag
    - GHCR package page shows `sha-<commit>` tag
    - No manually created secrets in repo settings
  </acceptance_criteria>

  <done>
