Nên dùng K8S hay Managed Service?
Service	Chạy trên K8S	Dùng Managed AWS	Khuyến nghị
RabbitMQ	Được, dùng Helm chart	Amazon MQ	Managed - ít ops hơn
Redis	Được	ElastiCache	Managed - xem bên dưới
ElasticSearch	Được	OpenSearch Service	Managed - scale dễ
MongoDB	Được	DocumentDB	K8S nếu cần tiết kiệm chi phí
PostgreSQL	Không nên	RDS	Managed - backup/failover tự động
GitLab Runner	Nên dùng K8S	GitLab.com SaaS	K8S - chạy job song song tốt
Rule of thumb: Stateful services (DB, cache, queue) → Managed AWS. Stateless (runner, app) → K8S.

Flow Chi Tiết Theo Tình Huống
1. User đăng nhập:


User → ALB → Account Service Pod
  → Redis (check session cache)
  → RDS PostgreSQL (query user)
  → Redis (lưu session)
  → trả token về User
2. User upload file:


User → ALB → File Service Pod
  → S3 (lưu file)
  → RabbitMQ (publish event "file uploaded")
  → Consumer Pod (xử lý async: resize, scan...)
  → MongoDB (lưu metadata file)
3. User tìm kiếm:


User → ALB → API Pod
  → Redis (check cache)
  → OpenSearch (nếu cache miss)
  → trả kết quả → Redis (cache lại)
4. Developer deploy code:


Push code → GitLab
  → GitLab CI Runner (build, test)
  → ECR (push Docker image)
  → EKS (rolling update pods)
  → CloudWatch (monitor logs/metrics)
